# 面向城市场景的视觉多模态目标检测：零基础导读

这份文档默认读者会 Python、深度学习和 agent 工具使用，但没有系统学过计算机视觉检测任务。目标是把这个赛题到底在考什么、我们现在为什么这样做、后面该往哪里改讲清楚。

## 1. 这个赛题在做什么

赛题名称是“面向城市场景的视觉多模态目标检测”。它不是图像分类，也不是图像分割，而是目标检测。

目标检测要回答两个问题：

1. 图像里有哪些目标。
2. 每个目标在哪里。

所以模型输出不是一句“这张图里有车”，而是一批框：

```text
class_id center_x center_y width height confidence
```

其中坐标是归一化坐标，范围一般在 0 到 1。`confidence` 是模型认为这个框可信的程度。

## 2. 什么叫多模态

这个赛题给的不是普通单张 RGB 图，而是三种图像信息：

```text
RGB：可见光图像，包含颜色、纹理、边缘等信息。
Infrared：红外图像，对热源、夜间、弱光场景有帮助。
Depth：深度图像，提供距离和空间结构信息。
```

赛题想让我们做的不是只拿 RGB 跑一个现成 YOLO，而是想办法把 RGB、红外、深度三种信息融合起来，让模型在城市复杂场景中更稳。

但这里有一个现实问题：多模态融合不等于“把三张图随便拼起来”。如果融合方式不好，红外和深度会变成噪声，反而拖累检测效果。

## 3. 评价指标为什么难

平台用的是 `mAP@50-95`。这比普通 `mAP@50` 更严格。

直观理解：

```text
mAP@50：
只要预测框和真实框重叠超过 50%，就可能算对。

mAP@50-95：
会同时考 50%、55%、60% ... 95% 多个重叠阈值。
框的位置越准，分数越高。
```

所以这个赛题不只看有没有检测出来，还看框画得够不够准。误检、漏检、框偏移都会扣分。

## 4. 提交文件有什么要求

每张测试图必须有一个同名 TXT 文件。即使没有检测结果，也要给空 TXT。

每行格式为：

```text
class_id norm_center_x norm_center_y norm_w norm_h confidence
```

注意点：

```text
1. 每张图最多 100 个预测框。
2. 类别不能非法。
3. 坐标不能越界。
4. confidence 不能缺失。
5. TXT 数量必须和测试图片数量一致。
```

我们当前的提交脚本已经能保证 1000 张测试图对应 1000 个 TXT 文件，并做格式校验。

## 5. 我们现在的基本路线

当前主线是：

```text
YOLO11M 检测器
+ RGB-guided-RDT 多模态预处理
+ TTA 推理增强
+ 置信度和 NMS 阈值校准
```

这里的 `RGB-guided-RDT` 可以理解为一种工程化多模态输入构造方式：仍以 RGB 为主，但把红外和深度信息转成模型能吃进去的辅助图像通道或融合表示。

这条路线的优点是稳定、训练快、提交可靠。缺点也很明显：它还不是很强的“特征级多模态融合”，所以提升空间有限。

## 6. 为什么最近只提升 0.6 分左右

原来平台最好大约是：

```text
50.0380
```

最近最好的包到了：

```text
50.6880
```

涨了约 0.65 分。这个提升主要来自推理后处理，不是模型结构突破。

我们发现：

```text
1536 高分辨率 + 极低 conf = 本地验证看起来涨，平台反而掉到 48/49。
1408 分辨率 + 更保守 conf = 平台更稳，最高到 50.6880。
```

这说明平台测试集上误检比较致命。置信度太低时，模型会输出大量低质量框，虽然本地验证可能吃到一点召回收益，但测试集会被误检拖下去。

简单说：我们最近是在“少报错框”上拿分，不是在“模型真正看懂更多目标”上拿分。

## 7. 目前最重要的经验

第一，不能盲信本地验证集。我们有几次本地验证上涨，但平台分数下降，说明本地验证集和测试集分布不完全一致。

第二，不能盲目用更大分辨率。1536 理论上能看清小目标，但也会放大噪声和误检。

第三，全量训练不一定更好。把 2000 张都拿来训练会导致没有干净验证集，本地分数会虚高，平台不一定涨。

第四，多模态不是报告里写得越复杂越好。真正有用的多模态融合必须能在平台分数上证明收益。

## 8. 如果想明显涨分，下一步应该做什么

仅靠调 `conf` 和 `iou`，大概率只能小幅提升。要想几分几分地涨，需要动到更根本的地方。

优先级建议如下：

| 方向 | 目的 | 风险 |
|---|---|---|
| 错误分析 | 找出哪些类别、哪些场景掉分 | 低 |
| 类别级阈值 | 不同类别使用不同 conf | 中 |
| 真实特征级多模态融合 | 让 IR/Depth 真正参与特征计算 | 中高 |
| 训练损失和样本重加权 | 针对小目标、难样本优化 | 中 |
| 伪集成式后处理 | 可能违反单模型精神，需要谨慎 | 高 |

我认为下一步最合理的是：先做错误分析和类别级阈值，再决定是否继续投入结构改造。

原因很简单：现在平台反馈已经说明误检控制很重要。与其继续盲目训练，不如先知道误检主要集中在哪些类别、哪些场景、哪些置信度区间。

## 9. 一句话总结

这个赛题表面是“多模态目标检测”，实际竞争点有三层：

```text
第一层：能不能稳定读数据、训练、提交。
第二层：能不能把 RGB/IR/Depth 融合得不拖后腿。
第三层：能不能针对平台指标控制误检、漏检和框质量。
```

我们现在第一层已经完成，第二层有工程化方案但还不够强，第三层靠阈值校准拿到了一点提升。后面想拉开差距，需要从“调提交包”转向“分析错误并让多模态真正帮助模型”。
