# 面向城市场景的视觉多模态目标检测阶段实验汇报

> 汇报时间：2026 年 6 月 15 日  
> 最好平台成绩：50.8190  
> 使用模型：YOLO11M + RGB-guided-RDT 多模态输入构造

---

## 1. 本阶段主要结论

本阶段完成了数据处理、模型训练、推理提交、平台反馈和单类别诊断。项目已有稳定基线，但模型级多模态融合还没有带来稳定增益。

目前表现最稳定的方案是：

```text
YOLO11M + RGB-guided-RDT 多模态输入构造
```

在标准训练/验证划分下，本地验证结果大致为：

```text
mAP@50     ≈ 0.772
mAP@50-95  ≈ 0.5005
```

平台成绩从早期稳定基线的：

```text
50.0380
```

提升到目前最好成绩：

```text
50.8190
```

累计提升约：

```text
0.7810
```

从实验现象看，分数提升主要来自推理阶段的后处理校准，尤其是类别级置信度阈值调整，而不是模型主体结构本身明显变强。模型已经具备一定检测能力，但仍存在低置信度误检、平台测试集与本地验证集分布不一致、多模态辅助信息利用不足等问题。

下一阶段不宜只围绕阈值做小幅调参，应优先弄清楚三个问题：

1. 本地验证集是否真的能反映平台测试集趋势；
2. RGB、Infrared、Depth 三种模态的数据质量、对齐和有效性是否可靠；
3. IR/Depth 到底在哪些场景下有帮助，怎样注入模型才不会变成噪声。

## 2. 赛题任务与评价指标说明

本项目对应的是城市场景下的视觉多模态目标检测任务。模型输入包括三类空间对齐图像：

| 模态 | 含义 | 可能提供的信息 |
|---|---|---|
| RGB | 可见光图像 | 颜色、纹理、边缘、普通视觉语义 |
| Infrared / IR | 红外图像 | 热目标、弱光或夜间环境下的目标轮廓 |
| Depth | 深度图像 | 目标距离、几何结构、遮挡和尺度信息 |

模型输出为每张测试图对应的 TXT 文件，每一行表示一个预测目标：

```text
class_id norm_center_x norm_center_y norm_w norm_h confidence
```

其中：

- `class_id` 表示类别编号；
- `norm_center_x / norm_center_y` 表示归一化后的目标框中心点坐标；
- `norm_w / norm_h` 表示归一化后的目标框宽高；
- `confidence` 表示模型对该预测框的置信度。

本赛题评价指标为 `mAP@50-95`。相比只看 `mAP@50`，它会在多个 IoU 阈值下综合评估模型表现，因此对目标框定位精度和误检控制要求更高：

```text
mAP@50     更关注“有没有大致检测到”
mAP@50-95  同时关注“有没有检测到”和“框得准不准”
```

因此，当前实验不能只看召回率，也不能只追求输出更多预测框。低置信度框虽然可能补充一部分漏检，但如果引入大量误检，平台分数反而会下降。

## 3. 实验变量与数字含义说明

### 3.1 数据划分

```text
1600/400
```

表示将官方训练集中的 2000 组样本划分为：

| 数字 | 含义 |
|---:|---|
| 1600 | 用于模型训练的图像组数 |
| 400 | 用于本地验证的图像组数 |

该划分用于观察模型在本地验证集上的 `mAP@50` 和 `mAP@50-95`，但后续实验表明，本地验证集趋势和平台测试集趋势并不完全一致。

### 3.2 输入尺寸

```text
1280 / 1408 / 1536
```

表示模型训练或推理时输入图像被缩放到的尺寸，通常指 YOLO 推理时的 `imgsz` 参数。

| 输入尺寸 | 含义 | 可能影响 |
|---:|---|---|
| 1280 | 早期常用输入尺寸 | 速度和精度较平衡 |
| 1408 | 后处理阶段平台表现较稳定的推理尺寸 | 相比 1280 保留更多细节，但误检相对可控 |
| 1536 | 更高推理分辨率 | 可能提升小目标召回，但也会产生更多低置信度候选框和误检 |

本阶段的实验现象是：`1536 + 极低置信度阈值` 在本地验证上看起来可能有提升，但平台分数反而明显下降；`1408 + 更保守的置信度阈值` 在平台上更稳定。

### 3.3 置信度阈值 conf

`conf` 是推理阶段保留预测框的最低置信度阈值。它控制模型输出框的数量。

| conf 较低 | conf 较高 |
| --- | --- |
| 输出框更多，可能提高召回 | 输出框更少，误检可能减少 |
| 也容易带来更多误检 | 也可能漏掉低置信度真目标 |

本阶段平台反馈说明，测试集对误检较敏感，因此过低的 conf 不一定有利。

### 3.4 NMS 阈值

NMS 是非极大值抑制，用于删除同一目标附近的重复预测框。NMS 阈值越低，重复框删除越严格；NMS 阈值越高，保留相邻框的倾向越强。

本阶段主要调整的是输入尺寸和置信度阈值，NMS 还没有做系统性类别级搜索，后续可以作为误检诊断的一部分。

## 5. 基线模型选择与稳定方案

### 5.1 为什么选择 YOLO11M

最初选择 YOLO 系列作为主干模型，主要基于以下考虑：

1. YOLO 系列工程生态成熟，训练、验证、推理和提交流程比较稳定；
2. 赛题标签本身就是 YOLO 格式，数据适配成本较低；
3. 训练数据规模只有约 2000 组，过大的模型不一定有优势；
4. YOLO11M 在模型容量、训练稳定性和显存消耗之间比较平衡。

稳定方案为：

```text
YOLO11M + RGB-guided-RDT 多模态输入构造
```

这里的 `RGB-guided-RDT` 指以 RGB 为主要视觉信息，同时引入 Infrared 和 Depth 构造多模态输入表示。它不是完整的三分支中期融合结构，但在验证和平台表现上最稳定。

### 5.2 稳定方案结果

在标准 `1600/400` 训练验证划分上，本地验证大致为：

```text
mAP@50     ≈ 0.772
mAP@50-95  ≈ 0.5005
```

平台早期稳定成绩为：

```text
50.0380
```

后续实验和提交包均以该方案作为主要基线。

## 6. 多模态融合实验

### 6.1 实验目的

赛题本身要求使用 RGB、Infrared 和 Depth 三种模态。理论上，红外可以补充弱光和夜间场景下的目标信息，深度图可以补充几何结构和距离信息。但多模态信息并不是简单加入就一定有效，如果对齐、归一化或融合方式不合适，辅助模态可能反而干扰 RGB 主干特征。

因此，本阶段围绕多模态输入和特征融合做了几类尝试。

### 6.2 已尝试方向

| 方向 | 目的 | 结果 |
|---|---|---|
| RGB 单模态基线 | 判断只用 RGB 的基础能力 | 作为对照组 |
| RGB-guided-RDT 输入构造 | 在输入层面引入 IR/Depth 辅助信息 | 表现最稳定 |
| IR/Depth 辅助 gate | 尝试让辅助模态以门控方式调制模型特征 | 未稳定超过基线 |
| 冻结主干只训练融合模块 | 避免破坏原 YOLO 主干能力 | 没有带来稳定提升 |
| 解冻 neck/head 后低学习率微调 | 让融合模块与检测头适配 | 仍未超过基线 |

最近一轮 IR/Depth gate 历史实验为：

```text
auxgate_rdtgate5_unfreeze_lr8e6_a002_e80_seed42_gpu0
```

其中：

- `unfreeze` 表示后期解冻部分网络参数；
- `lr8e6` 表示学习率约为 `8e-6`；
- `a002` 表示辅助融合强度系数约为 `0.02`；
- `e80` 表示训练 80 个 epoch；
- `seed42` 表示随机种子为 42。

该实验最好结果约为：

```text
mAP@50-95 ≈ 0.4979
```

低于稳定方案的约 `0.5005`，因此暂未作为正式提交模型。

### 6.3 判断

这个结果说明，IR/Depth 信息目前还没有通过简单 gate 结构转化为稳定收益。可能原因包括：

1. RGB、IR、Depth 虽然空间上对齐，但局部边界或目标区域仍可能存在偏差；
2. Depth 中存在无效深度区域，若处理不当会引入噪声；
3. 红外和深度在不同场景下可靠性不同，不应对所有图片、所有尺度采用同一融合强度；
4. 现有 gate 结构可能过于简单，没有区分不同尺度特征中辅助模态的作用。

如果继续做多模态结构，需要优先解决“何时信任 IR/Depth、在哪些尺度注入、如何避免辅助模态带来误检”的问题，而不是继续简单叠加输入通道。

## 7. 模型规模、小目标头与继续训练实验

### 7.1 更大 YOLO 模型复测

为了验证模型容量是否是瓶颈，复测了更大的 YOLO11L 和 YOLO11X。实验预期是：如果模型欠拟合，更大的模型可能提升检测能力。

但实际结果显示，YOLO11L / YOLO11X 在验证和平台表现上均不如 YOLO11M。可能原因是：

1. 训练数据规模较小，较大模型更容易过拟合；
2. 更大模型对 batch size 和学习率更敏感；
3. 多模态输入构造与大模型并不一定更匹配；
4. 主要瓶颈不在模型容量，而在误检、验证集失真和多模态融合方式。

因此，扩大 YOLO 模型规模不作为优先方向。

### 7.2 P2 小目标检测头实验

针对 `sign / ball / person` 等小目标或弱类，曾尝试加入 P2 小目标检测头。这里的 P2 指更高分辨率的浅层特征层，理论上对小目标更友好。

实验名称：

```text
p2head_lockedbest_lr1e5_e80_b4_seed42_gpu1
```

其中：

- `lockedbest` 表示从已锁定较优权重继续改造；
- `lr1e5` 表示学习率约为 `1e-5`；
- `e80` 表示训练 80 个 epoch；
- `b4` 表示 batch size 为 4；
- `seed42` 表示随机种子为 42。

该实验最好结果约为：

```text
mAP@50-95 ≈ 0.4021
```

明显低于主线。

这个结果不能说明“小目标头一定无效”。更合理的解释是：直接在已训练好的检测结构上增加 P2 head，会改变原有检测头和特征分配方式；如果没有重新设计训练策略、loss 权重和初始化方式，容易破坏原模型已经学到的稳定检测能力。小目标问题更适合先从小目标损失加权、定位损失优化或错误分析入手，而不是直接大幅改 YOLO head 结构。

### 7.3 继续训练与全量训练

基于最好权重，尝试过以下继续训练方式：

1. 在标准 `1600/400` 划分上进行低学习率微调；
2. 冻结部分主干后微调；
3. 使用全部 2000 张训练图进行最终训练。

实验结果显示，标准验证上的继续微调没有超过原基线，部分结果下降到约 `0.49`。全量训练虽然会让重合验证集分数变高，但平台提交后反而下降，说明这种验证分数不可作为可靠依据。

因此，继续训练不是主要收益来源。问题更可能来自测试集误检分布、类别阈值、验证集划分方式和模型泛化。

## 8. 推理后处理与平台反馈

### 8.1 后处理变量说明

推理阶段主要调整了以下变量：

| 变量 | 含义 |
|---|---|
| 输入尺寸 `imgsz` | 推理时图像缩放到的尺寸，例如 1280、1408、1536 |
| 置信度阈值 `conf` | 保留预测框的最低置信度 |
| NMS 阈值 | 删除重复框时使用的 IoU 阈值 |
| 类别级阈值 | 对不同类别单独设置不同的 conf 阈值 |

这些变量不会改变模型权重，但会影响最终提交的预测框数量和质量。

### 8.2 输入尺寸与全局阈值实验

最初尝试过更高输入尺寸 `1536` 配合极低置信度阈值。预期是通过更高分辨率提升小目标召回，通过低 conf 保留更多可能正确的候选框。

但平台结果下降到 48/49 左右，说明这种策略虽然可能提高部分召回，但引入了过多低质量预测框，误检代价更大。

之后改用 `1408` 输入尺寸，并采用更保守的置信度阈值，平台表现更稳定，并持续刷新成绩。这个结果说明，平台测试集上误检控制比单纯增加候选框更重要。

### 8.3 平台提交结果

| 阶段 | 设置或策略 | 平台结果 |
|---|---|---:|
| 早期稳定基线 | YOLO11M 主线 | 50.0380 |
| 全局阈值调整 | 1408 输入尺寸，conf=0.001 | 50.2380 |
| 更保守阈值 | 1408 系列阈值校准 | 50.6880 |
| 类别级阈值 A | 提高部分稀有类别阈值 | 50.75 |
| 类别级阈值 B | 稀有类别 + sign 调整 | 50.77 |
| 类别级阈值 C | 进一步提高 person 阈值 | 50.78 |

随后以 `50.78` 的提交包为基线，做了单类别诊断实验。这里的“单类别诊断”指每次只进一步收紧一个类别的置信度阈值，其他设置保持不变，以观察该类别阈值变化对平台总分的影响。

| 诊断包 | 改动 | 平台结果 |
|---|---|---:|
| sign003 | 提高 sign 类阈值 | 50.80 |
| bicycle002 | 提高 bicycle 类阈值 | 50.80 |
| car002 | 提高 car 类阈值 | 50.79 |
| light002 | 提高 light 类阈值 | 50.79 |
| animal002 | 提高 animal 类阈值 | 50.8010 |
| person003 | 提高 person 类阈值 | 50.8190 |

最好平台成绩为：

```text
50.8190
```

相比早期稳定基线：

```text
50.0380 -> 50.8190
```

累计提升约：

```text
0.7810
```

### 8.4 对后处理实验的理解

这些结果说明，模型低置信度预测框中存在一部分会明显影响平台得分的误检。提高部分类别阈值后，虽然可能损失少量召回，但减少误检带来的收益更大。

其中 `person003` 得分最高，说明 person 类阈值变化对平台总分比较敏感。但这不一定等于“person 类就是最大错误来源”，也可能是因为 person 类样本数量较多，阈值变化对总 mAP 的影响更明显。该判断仍需要结合本地错误可视化和 hard validation 进一步验证。

## 9. 已排除或暂缓的方向

为了避免重复投入，本阶段可以暂时排除或降低优先级的方向包括：

| 方向 | 结论 |
|---|---|
| 继续扩大到 YOLO11L / YOLO11X | 已验证不如 YOLO11M，暂不作为主线 |
| RT-DETR 主线 | 此前实验明显低于 YOLO11M，暂不继续投入 |
| 直接增加 P2 head | 现有实现明显破坏原结构，暂不直接采用 |
| 简单 IR/Depth gate | 没有稳定超过基线，需要重新检查数据和融合方式 |
| 低学习率继续微调 | 没有稳定收益 |
| 全量 2000 图训练并用重合验证集判断 | 验证分数失真，平台反而下降 |
| 1536 高分辨率 + 极低 conf | 平台误检增多，表现下降 |

这些失败或未超过基线的实验说明：瓶颈不是“模型不够大”或“训练轮数不够”，而更可能是评估体系、误检分布、多模态可靠性和定位质量问题。

## 10. 实验结论

阶段性结论如下。

第一，YOLO11M 仍然是最可靠的检测基座。更大的 YOLO 模型和 RT-DETR 没有带来稳定收益，因此不应继续盲目换模型。

第二，多模态信息还没有通过简单 gate 结构转化为稳定增益。IR/Depth 的价值仍然需要通过更严格的单模态、双模态、三模态对照实验来验证。

第三，推理后处理对平台分数影响较大。平台测试集似乎对误检比较敏感，过低置信度阈值和过高输入分辨率会增加低质量预测框，导致得分下降。

第四，本地验证集和平台测试集存在分布差异。单纯依赖随机 `1600/400` 划分得到的本地 mAP，不能完全代表平台趋势。

第五，提升主要来自对误检的抑制，而不是模型主体能力提升。若要取得更大幅度突破，需要从验证集构建、多模态可靠性、特征级融合和定位损失等方面继续推进。

## 11. 思考与请教

### 11.1 首先需要重建更可靠的验证体系

目前最大的问题之一是本地验证集与平台趋势不一致。随机 `1600/400` 划分可能会让相似场景同时出现在训练集和验证集，使本地验证结果偏乐观。

后续建议按场景、序列或拍摄来源重新划分验证集，并额外构建一个 hard validation。hard validation 可以专门包含夜间、弱光、遮挡、小目标、人群密集、深度无效、红外弱响应等困难样本，用来更真实地反映平台测试集风险。

### 11.2 第二步应检查三模态数据质量

IR/Depth gate 没有超过基线，还不能确定这是融合结构设计不合理，还是原始三模态数据在对齐、归一化和有效值处理上存在问题。

建议先做一轮三模态数据体检，包括：

1. 可视化 RGB / IR / Depth / Depth mask；
2. 检查 RGB 与 IR、RGB 与 Depth 的目标边界是否对齐；
3. 统计 Depth 的无效像素比例；
4. 统计 RGB 亮度、IR 对比度、Depth 有效区域等可靠性指标；
5. 将样本按低光、深度无效、红外强响应等场景分组。

### 11.3 第三步应做严格的模态消融实验

虽然做过多模态输入和 gate 实验，但还需要更系统地回答一个问题：IR 和 Depth 到底分别有没有帮助。

计划在同样模型、同样训练划分、同样推理设置下，对比：

| 实验 | 输入 |
|---|---|
| A | RGB-only |
| B | IR-only |
| C | Depth-only |
| D | RGB + IR |
| E | RGB + Depth |
| F | RGB + IR + Depth |

这样可以判断：

- IR 是否主要提升夜间或低光场景；
- Depth 是否对遮挡、尺度变化或小目标有帮助；
- 三模态一起使用是否存在互相污染；
- RGB-guided-RDT 是否已经吸收了主要多模态收益。

这一步会决定后续技术路线。否则直接做复杂融合，可能仍然不知道到底是哪一个模态在起作用。

### 11.4 第四步才是重新设计多模态融合结构

如果前面验证 IR/Depth 确实有用，结构上也不宜采用三套完整 backbone 平行融合。数据量较小，完整三分支模型过拟合风险较高。

更合理的方向可能是：

```text
RGB 主分支 + IR/Depth 辅助可靠性引导
```

也就是：

```text
RGB -> YOLO11M 主干
IR / Depth -> 轻量辅助分支
IR/Depth 特征 -> 可靠性 gate 或 prompt
在 P3/P4/P5 特征层调制 RGB 特征
```

这个思路主要参考官方论文中几类方法：

- YOLO 提供稳定的单阶段检测基座；
- CSSA 提供轻量特征级融合和空间/通道注意力思路；
- RDTTrack 提供 RGB 主模型 + Depth/Thermal 辅助 prompt 的思路；
- EvaNet 提供“先判断模态可靠性，再决定融合权重”的启发；
- BDDS 提供小目标定位和多尺度融合的启发。

需要请教的问题：在目前数据规模下，是否应该优先做轻量辅助融合，而不是完整三分支融合？

### 11.5 第五步是针对 mAP@50-95 的定位优化

稳定方案 `mAP@50 ≈ 0.772`，但 `mAP@50-95 ≈ 0.5005`。这说明模型并不是完全检测不到目标，而是高 IoU 阈值下定位精度仍然不足。

可以考虑：

1. 分析各类别 AP50 与 AP75/AP90 的差距；
2. 针对小目标做 loss 加权；
3. 借鉴 BDDS 中 DSSM loss 的思想，在 IoU 之外加入中心点距离、宽高差异、长宽比或对角线相似度等约束；
4. 优先关注 `uav / ball / light / sign / person` 等小目标或弱类。

需要请教的问题：如果加入定位辅助损失，是先只对小目标加权比较稳，还是直接修改整体 box loss 更合适？

### 11.6 后处理校准仍然有用，但不应作为唯一主线

平台最高分来自类别级阈值校准，说明后处理确实有效。但这更像是在现有模型上抑制误检，属于“挖掘现有模型潜力”，而不是根本提升模型能力。

后续可以继续做类别级 conf、NMS、面积过滤和宽高比过滤，但这些工作最好建立在 hard validation 和错误样例分析之上，而不是只根据平台总分反推。否则容易出现对平台反馈的过拟合。

---

## 12. 下一阶段拟解决的问题

结合以上分析，下一阶段可以把目标从“继续尝试新模型”调整为“找出模型误检和多模态无效的根因”。具体问题包括：

1. 随机验证集是否高估了模型泛化能力？
2. 平台上误检更敏感的类别和场景是什么？
3. RGB、IR、Depth 三模态是否存在局部对齐或有效值问题？
4. IR 和 Depth 分别在哪些场景下能带来增益？
5. 是否可以构建可靠性引导的轻量特征级融合模块？
6. 高 IoU 阈值下定位不准主要集中在哪些类别和尺寸范围？

如果这些问题能够解决，再进行结构设计会更有依据。否则继续增加模型复杂度或训练轮数，可能只是重复已经验证过收益有限的路线。

---

## 13. 阶段总结

本阶段的主要收获不是单纯把分数从 `50.0380` 提升到 `50.8190`，而是逐步明确了项目的真实瓶颈。

可以确认：

1. YOLO11M 是最稳定的检测基座；
2. 更大模型、继续训练、全量训练没有稳定收益；
3. 简单 IR/Depth gate 没有超过基线，说明多模态融合需要更谨慎；
4. 平台测试集对误检比较敏感，类别级后处理能带来小幅提升；
5. 本地验证集和平台趋势不一致，是实验判断中最需要解决的问题。

因此，下一阶段不再单纯围绕模型规模或训练轮数做尝试，而是先建立更可靠的验证与错误分析体系，再检查多模态数据质量，最后基于分析结果重新设计更有针对性的多模态融合和定位优化方案。
