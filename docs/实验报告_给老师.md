# 面向城市场景的视觉多模态目标检测阶段实验汇报

## 1. 任务概况

本项目面向城市场景下的多模态目标检测任务。赛题输入包括 RGB、Infrared 和 Depth 三类图像，输出为每张测试图对应的检测结果 TXT 文件。评价指标为 `mAP@50-95`，相比只看 `mAP@50` 的指标，它对目标框定位精度和误检控制要求更高。

赛题约束主要包括：仅使用官方训练数据；允许使用公开预训练权重；训练和推理均需离线完成；不采用多个模型简单投票、平均等集成方式；每张测试图必须生成同名 TXT 文件，且格式符合官方要求。

## 2. 工程和数据处理

前期已经完成完整工程闭环，包括：

```text
1. 官方训练集、测试集读取和结构检查。
2. RGB / Infrared / Depth 多模态数据预处理。
3. YOLO 格式标签读取、转换和校验。
4. 训练、验证、推理、提交包生成脚本。
5. 提交文件格式校验，确保 1000 张测试图对应 1000 个 TXT 文件。
```

由于本地和数据路径中包含中文，图像读写部分做了路径兼容处理，避免 OpenCV 在中文路径下读图失败。提交前均使用脚本检查 TXT 数量、类别、坐标和置信度字段。

## 3. 基线模型

最初选择 YOLO 系列作为主干模型，主要考虑其工程成熟、训练速度快、显存需求可控，适合当前数据规模快速迭代。

当前主线模型为：

```text
YOLO11M + RGB-guided-RDT 多模态输入构造
```

在标准 1600/400 训练验证划分上，主线模型本地验证大致为：

```text
mAP@50     ≈ 0.772
mAP@50-95  ≈ 0.5005
```

早期平台最好成绩为：

```text
50.0380
```

这一路线目前仍然是最稳定的基线。

## 4. 多模态融合尝试

围绕 RGB、Infrared、Depth 三模态输入，已经尝试过以下方向：

```text
1. RGB 单模态基线。
2. RGB-guided-RDT 多模态输入表示。
3. 基于 IR/Depth 的辅助 gate 特征融合。
4. 冻结主干、只训练融合模块。
5. 解冻部分 neck/head 后低学习率微调。
```

实验结果显示，简单增加 IR/Depth 辅助 gate 没有稳定超过主线模型。最近一轮较完整的 IR/Depth gate 训练为：

```text
auxgate_rdtgate5_unfreeze_lr8e6_a002_e80_seed42_gpu0
最好 mAP@50-95 ≈ 0.4979
```

该结果低于主线，因此暂未作为正式提交模型。

目前可以确认的是：红外和深度信息不能简单加入模型。若融合方式不合适，辅助模态会变成噪声，导致泛化下降。后续如果继续做多模态结构，应重点解决“何时信任 IR/Depth、在哪些尺度注入、如何避免辅助模态带来误检”的问题。

## 5. 更大模型和小目标头尝试

我们复测过更大的 YOLO11L / YOLO11X，但验证和平台表现均不如 YOLO11M 主线。主要原因可能是数据量较小，更大模型更容易过拟合，同时训练成本更高。

另外，针对 `sign / ball / person` 等小目标或弱类，尝试过 P2 小目标检测头：

```text
p2head_lockedbest_lr1e5_e80_b4_seed42_gpu1
最好 mAP@50-95 ≈ 0.4021
```

该结果明显低于主线，说明直接增加 P2 检测头破坏了当前已训练好的检测结构，不能作为有效改进方向。

## 6. 继续训练和全量训练

基于当前最好权重继续微调时，尝试过：

```text
1. 标准 1600/400 划分上的低学习率微调。
2. 冻结部分主干后的微调。
3. 使用全部 2000 张训练图的最终训练。
```

标准验证上的继续微调没有超过原主线模型，部分结果下降到约 `0.49`。全量 2000 张训练虽然会让重合验证集分数变高，但平台提交后反而下降，说明这种验证分数不可作为可靠依据。

因此目前判断：继续训练不是主要收益来源，当前瓶颈更可能在测试集误检分布、类别阈值和模型泛化上。

## 7. 推理后处理和平台反馈

后续主要围绕推理阶段的输入尺寸、置信度阈值和 NMS 阈值做校准。

关键发现如下：

```text
1536 分辨率 + 极低 conf：
本地验证看起来有提升，但平台下降到 48/49 左右。

1408 分辨率 + 更保守 conf：
平台更稳定，持续刷新成绩。
```

这说明测试集上误检代价较高。低置信度会输出大量候选框，虽然可能提高召回，但会引入更多错误检测。

主要平台结果如下：

| 阶段 | 设置或策略 | 平台结果 |
|---|---|---:|
| 早期稳定基线 | YOLO11M 主线 | 50.0380 |
| 全局阈值调整 | 1408, conf=0.001 | 50.2380 |
| 更保守阈值 | 1408 系列阈值校准 | 50.6880 |
| 类别级阈值 A | 稀有类别提高阈值 | 50.75 |
| 类别级阈值 B | 稀有类别 + sign 调整 | 50.77 |
| 类别级阈值 C | 进一步提高 person 阈值 | 50.78 |

随后做了单类别诊断实验，以 `50.78` 的包为基线，每次只进一步收紧一个类别，结果如下：

| 诊断包 | 改动 | 平台结果 |
|---|---|---:|
| sign003 | sign 阈值提高 | 50.80 |
| bicycle002 | bicycle 阈值提高 | 50.80 |
| car002 | car 阈值提高 | 50.79 |
| light002 | light 阈值提高 | 50.79 |
| animal002 | animal 阈值提高 | 50.8010 |
| person003 | person 阈值提高 | 50.8190 |

当前最好成绩为：

```text
50.8190
```

相比最初稳定基线：

```text
50.0380 -> 50.8190
```

累计提升约 `0.7810` 分。

## 8. 当前实验结论

目前比较明确的结论有：

```text
1. YOLO11M 主线仍是当前最可靠模型。
2. 更大 YOLO、继续微调、全量训练都没有带来稳定收益。
3. 1536 高分辨率在平台上没有体现优势，反而容易增加误检。
4. 当前有效提升主要来自误检控制，而不是模型结构本身变强。
5. 平台测试集和本地验证集存在差异，本地 mAP 不能直接代表平台趋势。
6. 单类别诊断显示 person 类低置信度误检可能是当前较重要的扣分来源。
```

从本地错误分析看，弱类主要包括：

```text
sign、bicycle、ball、person
```

但平台反馈说明，分数提升并不只取决于弱类 AP，而更受误检分布影响。尤其 `person003` 诊断包取得当前最好成绩，说明提高 person 类阈值能减少一部分对平台指标有明显影响的误检。

## 9. 后续建议

短期不建议继续盲目训练新模型。更合理的方向是继续做诊断型实验：

```text
1. 围绕 person 类阈值继续细分，例如 0.003、0.0035、0.004。
2. 在 person 最优阈值基础上，再组合 animal / sign / bicycle 的小幅调整。
3. 将平台反馈反推为“测试集误检分布”，建立比普通验证集更可靠的 hard validation。
4. 若后续再做多模态结构，先验证 RGB-only、RGB+IR、RGB+Depth、RGB+IR+Depth 的真实差异。
```

中长期如果要取得几分级别提升，不能只靠阈值。需要重新设计多模态融合方案，使 IR/Depth 能真正降低误检或改善弱类定位。目前已有的简单 gate 方案没有成功，说明后续结构设计要更谨慎。

## 10. 阶段总结

本阶段已经完成从数据处理、模型训练、推理提交、平台反馈到诊断实验的完整闭环。平台成绩从 `50.0380` 提升到 `50.8190`，说明后处理校准和类别级误检控制是有效的。

但当前提升幅度仍然偏小，主要原因是模型主体能力没有明显提高。现有收益来自对测试集误检分布的逐步校准，而不是多模态融合或检测结构带来的根本提升。

下一阶段应从“继续调参”转向“找出主要误检类别和场景”，再决定是否投入更复杂的多模态融合结构。

## 11. 需要讨论的问题

1. 当前平台只给总分，不给类别 AP。是否能通过其他渠道了解各类别得分或错误样例？如果不能，就需要继续用单类别诊断包反推。

2. `person003` 包目前效果最好，是否说明测试集中 person 类误检占主要扣分？还是因为 person 数量多，阈值变化对总分影响最大？这个需要继续用 `person=0.0035/0.004` 做小步验证。

3. 本地验证集和平台趋势不一致。是否需要重新按场景、序列或拍摄来源划分验证集，而不是随机划分？

4. 目前 IR/Depth gate 融合没有超过基线。是融合结构问题，还是数据中 IR/Depth 与 RGB 对齐、归一化、有效值处理仍有问题？这需要优先检查原始三模态图像的对齐质量。

5. 若后续要在技术报告中强调多模态创新，需要先拿到一个真实有效的多模态增益结果。否则报告中只能把多模态作为工程尝试，而不是主要得分来源。

6. 是否继续投入结构改造，还是先把 person、animal、sign、bicycle 的类别级阈值和错误分析做到极致？从当前平台反馈看，后者短期更可靠，但前者才可能带来更大幅度提升。
