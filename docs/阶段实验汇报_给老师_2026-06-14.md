# 面向城市场景的视觉多模态目标检测阶段实验汇报

## 1. 任务概况

本项目面向城市场景多模态目标检测任务。输入包括 RGB、Infrared 和 Depth 三类图像，输出为每张测试图对应的目标检测结果 TXT 文件。评价指标为 `mAP@50-95`，对目标框定位质量要求较高。

项目约束包括：只能使用官方数据集训练，允许使用公开预训练权重；推理过程必须离线完成；不采用多模型投票、平均等简单集成方式；每张测试图必须生成同名 TXT 文件。

## 2. 工程与数据处理

前期完成了完整工程框架搭建，主要包括：

```text
1. 官方训练集、测试集读取和结构检查。
2. RGB / Infrared / Depth 多模态数据预处理。
3. YOLO 格式标签转换与校验。
4. 训练、验证、推理和提交包生成脚本。
5. 提交文件格式校验，确保 1000 张测试图对应 1000 个 TXT。
```

考虑到本地 Windows 路径包含中文，图像读写部分采用了更稳妥的路径处理方式，避免 OpenCV 在中文路径下读图失败。

## 3. 基线模型

我们首先选择 YOLO 系列作为主干模型，原因是其工程成熟、训练速度快、单卡 24GB 显存可控，适合本赛题小规模数据快速迭代。

主线模型为：

```text
YOLO11M + RGB-guided-RDT 多模态输入构造
```

在 1600/400 的训练验证划分上，较稳定的本地验证结果约为：

```text
mAP@50     ≈ 0.772
mAP@50-95  ≈ 0.5005
```

该模型对应的早期平台最好分数约为：

```text
50.0380
```

## 4. 多模态方向尝试

围绕赛题的 RGB、红外、深度输入，我们尝试过以下方向：

```text
1. RGB 单模态基线。
2. RGB-guided-RDT 多模态输入表示。
3. 基于 IR/Depth 的辅助 gate 特征融合。
4. 冻结主干、只训练融合模块。
5. 解冻部分 neck/head 后低学习率微调。
```

实验结果显示，简单增加辅助门控或继续微调并没有稳定超过主线模型。部分实验在本地验证集接近基线，但平台反馈不理想。因此目前没有把这些结构替换为正式提交模型。

这个结果说明，红外和深度信息确实不能简单加入。融合模块如果没有足够数据支撑，容易引入噪声，导致模型在测试集上泛化下降。

## 5. 大模型与更大 YOLO 的尝试

我们复测过更大规模的 YOLO11L / YOLO11X 路线，但结果低于 YOLO11M 主线。主要原因可能是数据量较小，更大模型更容易过拟合，同时训练和调参成本更高。

因此当前没有继续把资源投入到盲目放大模型规模，而是回到更稳定的 YOLO11M 主线做校准和错误分析。

## 6. 继续训练与全量训练

我们尝试过基于当前最好权重继续微调，包括：

```text
1. 标准 1600/400 划分上的低学习率微调。
2. 冻结前层后的微调。
3. 使用全部 2000 张训练图的最终训练。
```

其中标准验证集上的继续微调没有超过原主线模型，部分结果下降到约 `0.49`。全量 2000 张训练虽然训练集重合验证分数较高，但平台提交后反而下降，说明这类结果不能作为可靠模型选择依据。

结论是：当前阶段继续训练不是主要收益来源，过拟合风险较大。

## 7. 推理后处理校准

最近主要收益来自推理阈值校准。我们比较了不同输入尺寸、置信度阈值和 NMS 阈值。

关键发现：

```text
1536 分辨率 + 极低 conf：本地验证有提升，但平台下降到 48/49。
1408 分辨率 + 较保守 conf：平台更稳定，并取得当前最好结果。
```

平台反馈如下：

| 设置 | 平台结果 | 结论 |
|---|---:|---|
| 原最好包 | 50.0380 | 早期稳定基线 |
| 1408, conf=0.001 | 50.2380 | 小幅提升 |
| 1408 系列某组更保守阈值 | 50.6880 | 当前最好 |
| 其他更激进或 NMS 变化包 | 48-50.49 | 不稳定 |

从预测框数量看，低置信度版本会产生大量候选框，容易放大测试集误检。当前最有效的方向是适当提高置信度，控制误检数量。

## 8. 当前结论

目前最可靠的结论是：

```text
1. YOLO11M 主线仍然是当前最稳模型。
2. 更大模型和继续训练暂时没有带来稳定收益。
3. 1536 高分辨率并未在平台上体现优势。
4. 平台分数更受误检控制影响，置信度阈值校准有效。
5. 本地验证集与平台测试集存在分布差异，不能只看本地 mAP。
```

当前最好平台分数为：

```text
50.6880
```

相比原最好结果：

```text
50.0380 -> 50.6880
```

提升约 `0.65` 分。

## 9. 后续计划

后续不建议继续盲目扩大模型或全量微调。更合理的路线是：

```text
1. 做预测结果错误分析，统计误检和漏检集中在哪些类别和场景。
2. 尝试类别级置信度阈值，而不是所有类别共用一个 conf。
3. 对小目标和高误检类别单独分析框尺寸、置信度分布。
4. 再决定是否进行真正的特征级多模态融合改造。
```

如果后续继续做结构改造，建议围绕“RGB 主分支 + IR/Depth 辅助 gate”的轻量特征融合，而不是大规模更换检测框架。这样既符合赛题多模态要求，也能控制过拟合风险。

## 10. 阶段总结

本阶段完成了从数据处理、模型训练、推理提交到平台反馈校准的完整闭环。实验表明，当前分数瓶颈不在于代码是否跑通，而在于模型对测试集误检的控制和多模态信息的有效利用。

目前我们已经获得一个相对稳定的 50.6880 提交结果。下一阶段应从错误分析和类别级后处理入手，再决定是否投入更复杂的多模态融合结构。
