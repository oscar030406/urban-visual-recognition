## 4. 五篇论文的方法总结

### 4.1 CSSA：Channel Switching + Spatial Attention

论文：

```text
Multimodal Object Detection by Channel Switching and Spatial Attention
```

核心方法：

- RGB 和 IR 两个 backbone 分别提取特征。
- 多个 backbone stage 后插入 CSSA 模块。
- Channel Switching：用 ECA 评估通道重要性，低权重通道由另一模态对应通道补偿。
- Spatial Attention：用 channel-wise average pooling 和 max pooling 生成空间权重。
- 融合特征送入 FPN 和检测头。

对本项目的结论：

- 后续必须做特征级融合。
- 不再把图像级融合当作最终创新主线。
- 可实现 `CSSA-lite`，作用于 YOLO P3/P4/P5 特征。

### 4.2 YOLO：检测基座

论文：

```text
You Only Look Once: Unified, Real-Time Object Detection
```

核心方法：

- 单阶段端到端目标检测。
- 一次前向同时预测 bbox、类别和置信度。
- 优点是工程稳定、速度快。
- 缺点是定位误差较明显，小目标和密集目标更难。

对本项目的结论：

- YOLO11M 继续作为主检测基座。
- 后续不再盲目换 YOLO11L/YOLO11X。
- 创新应集中在多模态特征融合和定位损失。

### 4.3 EvaNet：融合质量评估思想

论文：

```text
EvaNet: Towards More Efficient and Consistent Infrared and Visible Image Fusion Assessment
```

核心方法：

- 不是检测模型，而是红外-可见光融合质量评价方法。
- 将融合图分解成红外成分和可见光成分。
- 分别评估各模态信息保留程度。
- 引入环境分支，自适应调节可见光和红外权重。
- 原文使用 LLM 生成环境标签。

对本项目的结论：

- 不训练 EvaNet。
- 不使用 LLM，因为赛题禁止在线服务/API。
- 只借鉴“模态可靠性”思想，离线计算：
  - RGB 亮度和对比度；
  - IR 对比度；
  - Depth 有效像素比例；
  - Depth 梯度和无效 mask。

### 4.4 BDDS：FMAB + 双向融合 + DSSM Loss

论文：

```text
Learning Bi-Directional Fusion and Deformation-Sensitive Loss for RGB-T Tiny Object Detection
```

核心方法：

- FMAB：细粒度模态注意力。
  - shared branch 提取模态共有信息。
  - specific branch 提取各模态独有信息。
  - 消融显示一层卷积最好。
- Bi-directional Progressive Fusion：
  - 同时做 top-down 和 bottom-up 多尺度融合。
  - 减少信息丢失和信息污染。
- DSSM Loss：
  - 针对小目标定位。
  - 在 IoU 外加入中心点距离、宽高差异、对角线余弦相似度。

对本项目的结论：

- 后续应重点优化小目标 mAP@50-95。
- YOLO neck 本身有 FPN/PAN，可在 P3/P4/P5 插入轻量融合模块。
- 可实现 `DSSM-style` 辅助 box loss。

### 4.5 RDTTrack：RGB 主模型 + Depth/Thermal Prompt

论文：

```text
Collaborating Vision, Depth, and Thermal Signals for Multi-Modal Tracking: Dataset and Algorithm
```

核心方法：

- 输入 RGB + Depth + Thermal。
- 使用预训练 RGB-only 模型作为主模型。
- Depth 和 Thermal 先融合，并通过正交投影减少冗余。
- 融合后的 D/T 特征作为 prompt 注入 RGB 主模型。
- 冻结 RGB 主模型，只微调 prompt 和辅助融合模块。

对本项目的结论：

- 后续采用 `RGB 主分支 + IR/Depth 辅助 gate/prompt`。
- 不做三套完整 backbone 对等训练，数据太少，过拟合风险高。
- IR/Depth 先融合为辅助特征，再调制 RGB 主干。

## 7. 实验诊断

### 7.1 为什么 YOLO11M 仍是最佳

YOLO11M 在当前数据规模下最平衡：

- 参数量足够表达目标检测任务；
- 比 YOLO11L/X 更不容易过拟合；
- batch 和训练稳定性更好；
- 与 1280 输入和当前融合预处理匹配较好。

实验表明，继续扩大到 YOLO11L/X 没有带来收益。

### 7.2 为什么 RT-DETR 没有成为主线

RT-DETR-L 960 能跑通，但 `mAP@50-95=0.35044`，明显低于 YOLO11M。

RT-DETR-L 1280 基本失败，可能原因：

- batch=1，训练不稳定；
- DETR 系模型对小数据集和调参更敏感；
- 当前数据只有约 2000 组，端到端查询式检测器难以充分收敛；
- 与现有 `rgb_guided_rdt` 输入融合不匹配。

因此 RT-DETR 不再作为主线。

### 7.3 为什么当前最佳还不是“理想三分支融合”

当前最高分来自 `rgb_guided_rdt` 图像级融合，确实使用了 RGB、Infrared、Depth，但不是完整三分支中期融合。

真正模型级融合 `gate5` 已经实现：

```text
输入: [R, G, B, IR, Depth]
结构: RGB 主分支 + IR/Depth gate stem
```

但其最好结果约 `0.48520`，低于当前最佳 `0.50049`。

因此目前提交使用图像级多模态融合作为 fallback；后续创新主线必须转向特征级融合。

### 7.4 为什么下一步不是继续调参

当前已尝试：

- 不同 YOLO 模型尺度；
- 1280/1408/1536 输入；
- ECA；
- CSA；
- P2 小目标头；
- gate5；
- RT-DETR；
- 学习率、warmup、cosine、box loss 权重等。

多数结果集中在 `0.48-0.50`，说明单纯调参收益有限。

突破点应转向：

```text
更符合五篇论文的特征级多模态融合
+ 小目标定位损失
```

## 10. 后续锁定方向

后续算法名暂定：

```text
RDT-CSSA-YOLOM
```

含义：

```text
RDT: RGB 主模型 + IR/Depth 辅助 prompt/gate
CSSA: Channel Switching + Spatial Attention
YOLOM: YOLO11M 检测基座
```

### 10.1 主结构

```text
RGB / IR / Depth 三模态同步几何增强
RGB -> YOLO11M 主分支 stem + backbone
IR -> light stem
Depth -> depth_norm + valid_mask -> light stem
IR/Depth -> auxiliary fusion
auxiliary feature -> gate/prompt
RGB P3/P4/P5 + gate/prompt -> CSSA-lite fused P3/P4/P5
YOLO detection head -> class/objectness/box
DSSM-style auxiliary box loss -> 小目标定位约束
official exporter -> TXT/zip, max_det<=100
```

### 10.2 统一数据处理

后续所有实验必须固定以下处理，避免结构改动和预处理改动混在一起：

| 模块 | 统一做法 | 原因 |
|---|---|---|
| RGB | 保持当前 locked best 流程不变 | 先控制变量 |
| IR | 视为单通道热强度，做分位数裁剪和归一化；不做 RGB 式颜色增强 | 官方 IR 三通道视觉特征高度一致，不是真彩色 |
| Depth | 做有效范围裁剪、`depth_norm`、`valid_mask`；0 或过小值标无效 | 防止无效深度噪声进入 gate |
| 几何增强 | RGB/IR/Depth 完全同步 | 三模态空间对齐不能被破坏 |
| 光度增强 | RGB 可做 HSV/brightness/contrast；IR 只做轻微对比增强；Depth 不做颜色增强 | 异质模态不能共用 RGB 增广 |
| 分辨率 | 第一轮固定 1280 | 当前最佳和多数有效实验都集中在 1280 |
| batch | 有效 batch 目标不低于 16，不足时用梯度累积 | 避免小 batch 训练不稳定 |
| split | 固定 dev split，不再反复改 | 保证所有 ablation 可比较 |

### 10.3 优先实现模块

第一优先级：保底复现。

```text
完整复现 locked best，不改结构。
目标: 在固定 dev split 上复现到 0.497-0.503。
意义: 先证明新服务器、环境、数据和 evaluator 可信。
```

第二优先级：AuxGate。

```text
RGB 主分支 + IR/Depth 轻量 stem + gate/prompt 注入 P3/P4/P5。
训练: 先冻结主干 10-15 epoch，只训辅助模块和检测头，再低学习率联合微调。
目标: 稳定超过 0.503。
```

第三优先级：AuxGate + CSSA-lite。

```text
在 P3/P4/P5 上加入轻量 channel switching + spatial attention。
目标: 稳定超过 0.510 或至少达到可解释的 0.003+ 提升。
```

第四优先级：AuxGate + CSSA-lite + DSSM-style。

```text
在 box loss 里加入中心距离、宽高差、对角线余弦相似度等小目标定位约束。
lambda 只搜 {0.05, 0.1, 0.2}。
目标: 总 mAP@50-95 不降，小目标 AP 明显提升，最终稳定超过 0.512。
```

第五优先级：FMAB-lite。

```text
只在前面方案稳定提升后再做。
目标: shared/specific feature 增强复杂场景召回。
风险: 参数和过拟合风险增加，不能优先做。
```

目的：

- 专门提升小目标定位质量。
- 观察是否提升 `mAP@50-95`，而不是只提升 `mAP@50`。

限制：

```text
lambda_dssm 只搜 {0.05, 0.1, 0.2}；
必须额外记录 small-object AP 和 per-class AP；
如果总 mAP 下降，即使小目标指标上升也不锁包。
```

### 暂停方向

本轮明确暂停：

- YOLO11L / YOLO11X。
- RT-DETR-L。
- 直接 `gate5` 五通道堆输入。
- 只做孤立 ECA / CSA / P2 微改。
- 开放式大调参、多尺寸乱试。
- 多模型集成、投票、平均、WBF 主线化。

## 12. 风险与控制

主要风险：

- 数据量小，完整三分支网络容易过拟合。
- 改 YOLO 内部 loss 和 neck 容易引入训练不稳定。
- 多模态模块如果太重，会损害已有最佳模型。
- 如果 split、evaluator、exporter 不统一，离线涨分可能没有线上意义。

控制策略：

- 不做三套完整 backbone。
- 不再盲目换大模型。
- 每个模块单独 ablation。
- 只有验证集超过 `0.50049` 才生成新锁定提交包。
- 保留当前最佳 zip，不被覆盖。
- 所有候选方案必须记录 seed、配置、epoch 曲线、best.pt、submission hash。
- 候选最佳必须做双种子复验，提升低于 0.003 不视为有效突破。

## 13. 最终判断

项目现在不缺 baseline，也不缺更大模型实验；真正的问题是：

```text
当前最佳是图像级多模态融合，
但赛题和五篇参考论文更支持特征级多模态融合。
```

因此后续路线锁定为：

```text
YOLO11M 主检测器
+ RGB 主分支
+ IR/Depth 辅助 gate/prompt
+ CSSA-lite 特征融合
+ DSSM-style 小目标定位辅助损失
+ 有条件再加 FMAB-lite 共享/特异特征增强
```

这条路线最符合赛题要求，也最符合当前实验事实。
