# 新服务器交接信息

本文档用于在新项目中复用当前新服务器。默认使用普通账号 `oscar`，不要使用 `root` 账号跑训练任务。

## 连接信息

```bash
ssh -i C:\Users\oscar\.ssh\codex_shuhuai_oscar_ed25519 -o StrictHostKeyChecking=no oscar@192.168.6.3
```

推荐在 Codex / PowerShell 中使用：

```powershell
ssh -i C:\Users\oscar\.ssh\codex_shuhuai_oscar_ed25519 -o BatchMode=yes -o StrictHostKeyChecking=no -o ConnectTimeout=12 oscar@192.168.6.3 "hostname && nvidia-smi"
```

说明：

- 内网地址：`192.168.6.3`
- 登录账号：`oscar`
- SSH key：`C:\Users\oscar\.ssh\codex_shuhuai_oscar_ed25519`
- 不建议在项目文档中保存明文密码。
- 服务器通过蒲公英网络访问，本机需要先接入对应网络。

## 目录约定

所有新项目都放在：

```bash
/data/oscar
```

新项目建议格式：

```bash
/data/oscar/<project-name>
```

不要占用或覆盖已有目录：

```bash
/data/oscar/global-campus-ai-algorithm
```

创建新项目目录示例：

```bash
mkdir -p /data/oscar/new-project-name
cd /data/oscar/new-project-name
```

## GPU 信息与使用规则

服务器有两张 GPU：

```text
GPU0: NVIDIA GeForce RTX 4090 D, 约 24GB
GPU1: NVIDIA GeForce RTX 4090 D, 约 24GB
```

GPU0 常驻有 root 服务，大约占用 3.9GB 显存，通常不要杀：

```text
/opt/conda/bin/python ... gunicorn
python3 app_main.py
```

使用策略：

- 不妨碍别人已有任务。
- 可以尽量用满剩余显存，但不要 OOM。
- 启动训练前先看 `nvidia-smi` 和当前进程。
- 不要杀 root 服务。
- 并行训练前确认每个任务绑定到不同 GPU。

常用检查命令：

```bash
nvidia-smi
nvidia-smi --query-gpu=index,name,memory.total,memory.used,memory.free,utilization.gpu --format=csv,noheader,nounits
ps -u oscar -o pid,stat,etime,pcpu,pmem,cmd
```

查看训练/推理进程：

```bash
ps -u oscar -o pid,stat,etime,cmd | grep -E 'train|python|predict|infer|val' | grep -v grep
```

## Python 环境建议

每个项目独立建虚拟环境，避免污染其他项目：

```bash
cd /data/oscar/<project-name>
python3 -m venv .venv
source .venv/bin/activate
python -m pip install -U pip
```

如果项目需要 PyTorch/CUDA，优先根据项目需求自行选择版本。不要默认升级系统环境。

已有视觉项目曾使用：

```text
Python 3.12
torch 2.6.0 + cu124
ultralytics 8.4.60
```

这只是历史参考，不是新项目必须固定的版本。

## 文件传输

从本机上传到服务器：

```powershell
scp -i C:\Users\oscar\.ssh\codex_shuhuai_oscar_ed25519 -r "D:\local\path" oscar@192.168.6.3:/data/oscar/<project-name>/
```

从服务器下载到本机：

```powershell
scp -i C:\Users\oscar\.ssh\codex_shuhuai_oscar_ed25519 oscar@192.168.6.3:/data/oscar/<project-name>/outputs/result.zip "D:\local\output\"
```

大量数据建议用 `rsync`，如果 Windows 本机没有可用 rsync，则用分批 `scp`。

## 后台任务

简单后台启动：

```bash
nohup .venv/bin/python train.py --device 0 > logs/train_gpu0.log 2>&1 &
```

查看日志：

```bash
tail -f logs/train_gpu0.log
```

停止自己的任务：

```bash
kill <pid>
```

不要使用：

```bash
kill -9 -1
pkill python
```

这类命令可能误杀其他任务或服务。
